## What We Discussed — Full Summary

### 1. Recording Guide: How We Run Migrations with mxCLI

We're planning a demo recording showing the full migration pipeline. Key topics:

**The pipeline (what actually happened):**
1. Omnext delivered 114 OutSystems 11 eSpace XML blueprint files
2. Claude built the extraction infrastructure (xml-extractor.js, cs-extractor.js,
   doc-extractor.py, merger.js, run.js) in conversation
3. `node run.js 2` ran the extractors → produced 800k-line JSON knowledge base
   (screens.json, logics.json, entities.json, structures.json etc.)
4. Claude processed the Share/ Japanese business documents session by session
   (PPTX, xlsx via ZIP extraction, PDFs via pdftotext) → wrote KB_*.md files
   (tracked in Maurits-docs/decisions/EXTRACTION_LOG.md)
5. Claude synthesised the BRD JSON files (F001-F012) by reading extracted JSON + KB files
6. Claude produced enriched domain design docs (docs/design/F001-F012.md)
7. MDL scripts written against those design docs → built in Mendix via mxcli

**Key corrections from research:**
- The BRD JSON files were NOT generated by an automated agent/subprocess —
  they were produced by Claude in conversation, reading the extracted data
- The extractors were built by Claude in conversation (not pre-existing tools)
- The Omnext folder contains only XML files — no JSON blueprint file exists there
- `run.js` Phase 3 generates MDL scripts, NOT BRDs. BRDs were conversation output.

### 7. E2E Testing Approach (methodology, not yet a skill file)

What we developed across sessions:
1. Widget discovery first — DOM inspect / discover-widgets.js to build widget map per page
2. Suite structure: smoke → empty-submit → partial-fill → happy-path → robustness (E2E-01 to 05)
3. helpers.js pattern: shared login, dbQuery(), makeReporter(), .mx-name-* selectors
4. DB assertions: Mendix table naming (module$entity lowercase), FK truncation at 63 chars,
   PostgreSQL direct via psql.exe (M2EE token is randomised in Mendix 11.10+ → OQL blocked)
5. Bug format: symptom → evidence → root cause → suggested fix, never implement without approval
6. Gap report after each run: structured summary of passed/failed/blocked
7. Rule: never run two Playwright sessions simultaneously (Mendix session limit)
